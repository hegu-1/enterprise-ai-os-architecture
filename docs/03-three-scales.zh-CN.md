# 03 · 一个架构，三个尺度

> [English (full paper) →](../README.md) · [← 中文总览](../README.zh-CN.md)

同一套栈出现在三个尺度：n=1（一个人的记忆系统）、创业公司（一套运营 OS）、大厂（公司级 AI OS）。**不变的是内核，变的只是规模与合规。**

| 维度 | n=1 · 个人 | 创业 · ops-OS | 大厂 · 企业 |
|---|---|---|---|
| 感知 | 手喂 + session | 多渠道扫描 | 全系统 event 平台 |
| 智能 | 一个 render head | 模型 + 规则层 | 模型层 + 路由 + 微调 |
| 行动 | 手 + 几个脚本 | CLI + 编码 agent | 工具网关 + MCP farm |
| 记忆 ★ | markdown vault | 归档 + context | 知识图（role-scoped） |
| 协作主体 | 一人 + 多 AI render head | 一人搭 + 团队跑 | 全公司 + agent mesh |
| 权限 | 隐式 | 岗位/团队 | 组织 RBAC + 合规 |
| 治理 ⑦ | token + 自审 | dry-run + 人在环 | 监管 gate + 审计平台 |
| 反 silent-capture | 决策自己 author | 决策人审 | 决策人 ratify + provenance 强制 |

## 不变量（这就是内核）

1. **四层骨架**：感知 / 智能 / 行动 / 记忆——形态不随规模变。
2. **记忆即容器**：连续性住在记忆里、不住在模型里；模型可换。
3. **provenance**：谁/什么写的都留痕。个人尺度是给每个节点标作者，企业尺度是每节点强制字段——同一机制。
4. **人 authored 决策 vs agent 写连接组织**：反 silent-capture。个人尺度护连续性，组织尺度护问责。
5. **相位对齐 / 降低信息损耗**：个人 = 任何模型加载后从你的框架开始想；组织 = 信息不卡在岗位/BU 墙之间。同一个 telos。
6. **护城河只在 ③记忆 + ⑦治理**：三个尺度都成立；其余层是 commodity。

## 变量（只有这些随规模变）

- 脚本 → 平台 → 平台的平台（PaaS）
- 一个 render head → 多 AI 协作 → agent mesh + A2A
- 隐式权限 → 岗位 → 组织 RBAC + 合规
- 自审 → 人在环 → 监管 forcing function
- 几百节点 → 万级 → 千万级

## 关键

**变量都是「规模 / 合规」维度，不变量都是「内核」维度。** 机制错了，规模放大只会更快错；机制对了，剩下是工程与合规。

所以这不是「个人项目」和「大厂架构」两件事，是**同一件事在不同尺度**。路径是从 n=1 验证机制 → 向上放大，而不是从大厂往下抄。
