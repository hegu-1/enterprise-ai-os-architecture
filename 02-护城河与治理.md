# 02 · 两条护城河与治理内核

## 两条护城河

五个 commodity 层会在各厂商之间卷到同质——人人都有模型、协作 surface、云。差距在**记忆**和**治理**拉开。

- **记忆是护城河**，因为**认知会积累、不可复制**。
- **治理是护城河**，因为**可信的自治是目前没人敢安全交付的东西**——而信任基础设施随审计历史复利累积、与监管对齐（先行者定标准）、且与记忆层纠缠（provenance 住在记忆里）。两层咬死，一起不可复制。

## 治理内核（最难的不是能力，是治理）

企业 AI OS 最难的问题不是能力，是**自进化 agent + 持久记忆会漂移**——漂向陈旧判断、silent capture（组织的「记忆」悄悄偏离人真正决定的东西）、不可审的 mutation。张力是**速度 vs 控制**。解法是一个内核：保住速度，用 provenance 消灭漂移里**「悄悄」**那部分。

六条原语：

- **provenance-enforced** —— 每个 mutation 强制 emit 来源（谁/什么/为何/源）。不可协商；这本身就是审计轨迹，by construction。
- **drift-aware** —— agent 内置仪表，检测 execution-overrun / abstraction-drift / premature-closure，**surface 而非 suppress**。
- **judgment-aware** —— 持久记忆里区分**当前判断 vs 陈旧观点**；判断会过期。
- **schema-coexistence（core/edge）** —— 稳定判断在慢、人 ratify 的 **core**；演进能力在快、agent 驱动的 **edge**；edge→core 的晋升由 provenance 中介。
- **calibration-loop** —— 外部反馈 → tagged source → schema delta → **人 ratify** → audit trail。组织「学习」而不漂移的机制。
- **capability token** —— 越界动作要 scoped、过期、留痕的 token；**人持钥匙**；无 token 则 refuse + surface。

企业还要加：

- **risk-tiered judgment gate** —— 按 风险 × 可逆性 × blast-radius 路由：低风险自动 / 中风险人 ratify / 高风险多方会签。避免 over-gating 把自治的速度收益吃光。
- **合规映射** —— 政策与法规编译成机器可校验的规则，agent 行动前先过，审计证据自动生成。
- **可观测 / eval** —— 数千 agent 的持续评估、成本控制、异常检测。
- **授权链** —— 哪个 agent 以谁的 authority 行动，可吊销。
- **事故回滚** —— 每个 action 可逆或可补偿；provenance 支撑取证与回滚。

## 深层原理

内核是**人类判断 与 自进化 agent 之间那层缺失的内核**。它的作用：让组织**部署自治 agent 而不丢失人类问责**。没有它，没有任何严肃的企业敢把 agent 放开跑——这正是为什么**治理（而非能力）才是真门槛**。

- 个人尺度：让「某天 AI 替我决定」是 authored（被授权、可逆可审、被宣告），不是 drifted。
- 组织尺度：让「全公司放 agent 自动跑」是**可问责的自治**，不是失控。

同一个内核关切，actor 从 1 变成全公司。
